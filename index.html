Reward Squad

Loyalty Points PWA

  • Progressive Web Application
  • Django / Python
  • JavaScript, HTML & CSS
  • MySQL

As part of 3rd year in TU Dublin, we were tasked to design and build a software application. After research, I determined to implement a Loyalty Points system designed to integrate with eCommerce websites e.g. OpenCart, WooCommerce, etc. The programme was designed modularly, allowing it to be adapted for many different frameworks. The application also features a self-checkout system, allowing users to scan products with their smart phone to get more information (e.g. price) on the product, and add them to an order. A member of staff must authorise the checkout, helping prevent theft.

The application has several user types (customer, clerk, merchant, and administrator), each with their own view (showing them the most relevant information). Built to accommodate multiple stores, Reward Squad facilitates the users to quickly and easily change store, or to enroll with a new store. It is completely contactless, employing QR codes and scanners for both staff and customers alike.

Customers can earn and spend reward points, online or in store, and have them shared across two distinct systems. This was achieved by robust transaction control

You can view the source code on GitHub

ESERO - Micro:bit

CSInc

  • C#
  • TypeScript
  • WinForms

In the Summer of 2021, I enrolled in the CSInc programme, to help give back to the community. Working with Keith Quille (my former Software Development lecturer) of TU Dublin, and a small team, we liased with the ESERO team, and another CSInc team to develop software that would allow secondary school students to more easily work with the Micro:bit for the ESERO CanSat challenge.

The challenge was to abstract as much from the user as possible - when the Micro:bit is connected, the software will scan all ComPorts to detect it. If not found, it will present a message to the user. Taking feedback from the other CSInc team (tasked with writing the syllabus for the course), we implemented a helpful graphic to illustrate the different Micro:bit versions (V1 and V2).

The two primary components of this development were A) the TypeScript programme to be installed onto the Micro:bit to communicate with the computer, and B) the C#/WinForm application to display the sensor data to the user. The C# programme allows the user to record the sensor data and export to CSV for analysis. The V1 and V2 Micro:bits have different sensors, and so the programme was designed in a way to hide sensor data not present (in order to remove any confusion).

This software is not yet hosted on GitHub, however it is in use with schools around the country, helping get the younger generation interested in Computer Science.

Human Centred AI Masters

Website Migration

  • Wordpress
  • AWS Linux 2
  • Bootstrap 5
  • PHP, JavaScript, HTML & CSS

This project is currently ongoing. As project lead, my job is to identify and assign tasks, as well as overseeing the whole process.

Working with TU Dublin, the Budapest University of Technology & Economics, University of Applied Sciences Utrecht, and University Federico II Italy, the task is to port their static HTML/CSS website to a content management system (CMS). Working as a team of two, plus a liason between the Universities, Wordpress was identified as the most suitable CMS, and data migration has started.

The project is broken up into phases - 1.) installation & configuration of Wordpress, 2.) data migration & design, and 3.) improved functionality. The current design is to be adapted to Wordpress, with minimal changes, instead utilising the improved functionality Wordpress offers. This functionality will be extended as the project continues, with the final phase being maintenance.

Under the hood, work is continuing to automate the process to facilitate greater failsafes, high availability, and smoother transitions to other hosing services
